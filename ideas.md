# Идеи из Вастрик.Клуба — фит для $mol + Giper Baza

## Tier 1 — Идеальный фит (5/5)

### 1. Метки на карте — совместный сервис закладок на карте (70 комментов)

**Суть:** У людей копятся сотни флажков на Google Maps — бары, кемпинги, автомойки, красивые места. Гугл даёт всего 3 типа меток, они не расшариваемы нормально, нет категорий. Нужен сервис где можно организовать метки по спискам/тегам и шарить их с друзьями. Типа «мои любимые бары в Берлине» — отправил ссылку другу и он видит карту.

**Почему $mol+GB:** Real-time совместное редактирование пинов на карте — это прямо то, для чего GB создан. Каждый юзер владеет своими метками (P2P), но может расшарить списки. Офлайн-first критичен (путешественники без интернета). $mol даёт реактивный UI поверх карты.

**Конкуренты:** Google My Maps (убогий, 3 типа), uMap (open-source, но без P2P), Mapotic, Organic Maps (нет шеринга). Ни один не даёт P2P офлайн-синк между друзьями.

**Вердикт: BUILD** — понятный MVP, большой спрос, нет конкурента с P2P.

---

### 2. Трекинг посещённых улиц — туман войны на карте (41 коммент)

**Суть:** Карта города изначально серая (как fog of war в играх). По мере того как ходишь/ездишь по улицам, они раскрашиваются. Видно какой процент города ты исследовал. Автор переехал в СПб, хотел изучить окрестности на самокате — нашёл только Fog of World на iOS (платный, без веб-версии). Нет нормального кросс-платформенного решения.

**Почему $mol+GB:** GPS-трекинг — личные данные, офлайн-first (гуляешь где нет сети, потом синкается). Киллер-фича: P2P шеринг прогресса с друзьями — «смотри, я открыл 47% Питера, а ты?» Геймификация через GB.

**Конкуренты:** Fog of World (iOS only, $4, нет социалки), Wanderlust (мертвый), City Wanderer. Ни у кого нет P2P синка прогресса между друзьями.

**Вердикт: BUILD** — уникальная социальная фича, веб = доступнее чем iOS-only.

---

### 3. Где мои друзья — карта локаций друзей (19 комментов)

**Суть:** В чатах тусовок-экспатов постоянно: «а ты сейчас в каком городе?», «есть кто в Барселоне?». Каждый раз один вопрос адресован одному человеку, но читают все. Нужен бот/карта где видно кто из закрытого комьюнити где находится. Автор предлагает Telegram-бот + веб-карту.

**Почему $mol+GB:** Локация — максимально чувствительные данные. Zenly (Snap) закрылся, Life360 пойман на продаже данных брокерам. P2P через GB = данные не хранятся на сервере, только между друзьями. Реактивное обновление позиций.

**Конкуренты:** Life360 (скандал с продажей данных), Glympse, Google Maps sharing — все централизованы. Zenly закрыт. Приватной P2P альтернативы нет.

**Вердикт: BUILD** — рынок есть (Zenly имел 40M юзеров), P2P = настоящий дифференциатор.

---

### 4. Распределённые блоги (23 коммента)

**Суть:** Автор изучал P2P технологии (ZeroNet, Tox, DHT, I2P) и предлагает концепт блогов где каждый пост принадлежит автору, контент распространяется между читателями P2P, нет центрального сервера. Никто не может забанить/удалить/монетизировать чужой контент.

**Почему $mol+GB:** Giper Baza буквально ЭТО — P2P синк данных между пользователями, каждый владеет своими записями. $mol — реактивный фид. Можно сделать как proof-of-concept чтобы показать возможности стека.

**Конкуренты:** Mastodon (федеративный, не P2P), Nostr (крипто-centric, плохой UX), Scuttlebutt (умер). У $mol+GB лучше эргономика чем у всех.

**Вердикт: BUILD** — идеальный шоукейс технологии, даже если не взлетит как продукт.

## Tier 2 — Очень хороший фит (4/5)

### 5. Менторы и падаваны — маркетплейс менторства (381 коммент!)

**Суть:** Самая популярная идея клуба. Люди пишут чему хотят научиться, чему могут научить, и что предлагают взамен. Ищут пары ментор-ученик. Сейчас это просто тред в комментариях — неудобно, не масштабируется, нет поиска по скиллам.

**Почему $mol+GB:** Реактивный граф доступности (ментор отметил себя свободным → ученик видит мгновенно). Заметки с сессий, трекинг прогресса, скилл-матрица — всё collaborative через GB. $mol для реактивного поиска/фильтрации.

**Конкуренты:** MentorCruise ($150+/мес), GrowthMentor, Qooper — все корпоративные/платные. Для комьюнити ничего нет.

**Вердикт: BUILD** — максимальный спрос (381 коммент!), но скоуп большой.

---

### 6. Read-it-later который работает (28 комментов)

**Суть:** Все существующие сервисы «читай потом» (Pocket, Instapaper) бесят: не умеют грабить страницы за логином, непредсказуемо делают предзагрузку для офлайна. Часто обнаруживаешь что статья закинутая пару дней назад не загрузилась. Нужен нормальный ридер с надёжным офлайном.

**Почему $mol+GB:** Список чтения — личные данные, синк между устройствами P2P (не через чужой сервер). Офлайн-first — главная фича. **Pocket закрывается** — рыночное окно прямо сейчас.

**Конкуренты:** Pocket (закрывается!), Instapaper (заброшен), Readwise Reader ($8/мес), Raindrop.io. Окно возможности.

**Вердикт: BUILD** — Pocket умирает, рынок голый, P2P синк = конкурентное преимущество.

---

### 7. Семейные дела с подкреплением (28 комментов)

**Суть:** Семейной паре сложно делать домашние дела — всегда находятся более важные занятия. Идея: приложение где за выполненные дела (починил замок, вынес мусор, помыл окна) получаешь плюшки от партнёра (торт, поход в кино). Геймификация домашних обязанностей.

**Почему $mol+GB:** Общий список задач семьи с реактивными апдейтами (ребёнок сделал → родитель видит мгновенно). Офлайн (у детей не всегда есть wifi). P2P синк между членами семьи без сервера.

**Конкуренты:** OurHome, Habitica, S'moresUp, Joon — все централизованные. Ни один не P2P.

**Вердикт: BUILD** — простой скоуп, понятный MVP, офлайн P2P = реальное преимущество для семей.

---

### 8. Flashcards на базе Anki + коммьюнити (26 комментов)

**Суть:** Автор учит английский и китайский, пришёл к выводу что Anki — лучший инструмент для запоминания, но у него ужасный интерфейс и нет нормального шеринга колод между учениками. Хочет сделать современный Anki с красивым UI и сообществом для обмена колодами.

**Почему $mol+GB:** Колоды = данные юзера, P2P синк между устройствами (учишь на телефоне в метро, продолжаешь на ноутбуке). Шеринг колод между учениками через GB. Офлайн-first критичен (метро, самолёт).

**Конкуренты:** Anki (мощный но уродский UI, нет нормального синка), Quizlet (централизован, урезал бесплатный план), Memrise, Brainscape.

**Вердикт: BUILD** — Anki = #1 по функционалу но UX из 2005, P2P шеринг колод = киллер-фича.

---

### 9. Карта стран друзей (23 коммента)

**Суть:** Автор прошёл 2 релокации, впереди ещё одна. В каждом месте встречал экспатов/номадов которые часто переезжают. Хочет карту где отмечены страны/города где жили и живут друзья, коллеги, родственники. Чтобы перед поездкой в Лиссабон увидеть «о, у меня там 3 друга!».

**Почему $mol+GB:** Каждый юзер владеет своей историей перемещений (GB). Друзья подписываются друг на друга P2P. Реактивное обновление: друг переехал → на карте видно сразу. Простой скоуп для MVP.

**Конкуренты:** Нет доминирующего конкурента. Google Maps sharing не про это. Scattered ручные инструменты.

**Вердикт: BUILD** — маленький скоуп, нет конкурентов, полезно для экспат-комьюнити.

---

### 10. Как не дать идеям проектов пропасть (25 комментов)

**Суть:** Во время работы/прогулок приходят «гениальные» идеи (бот для Slack, новый тул). Записываешь в Obsidian/заметки — и забываешь. Нет системы чтобы идеи развивались: голосование, обсуждение, поиск единомышленников, трекинг прогресса.

**Почему $mol+GB:** Совместная доска идей с голосованием и тегами — реактивное обновление через GB. Мета-ирония: построить этот сервис на $mol+GB для отслеживания идей которые делать на $mol+GB.

**Конкуренты:** Notion, Coda, Airtable — тяжёлые и не про коллаборативные идеи. Нет лёгкого community-oriented инструмента.

**Вердикт: BUILD** — хороший внутренний инструмент + шоукейс.

---

### 11. Сервис коллективных действий (21 коммент)

**Суть:** Координация группы людей для совместного действия: «если 50 человек подпишутся, все идём на митап/покупаем курс/делаем проект». Проблема доверия: никто не хочет быть первым. Нужен Kickstarter но для действий, не денег.

**Почему $mol+GB:** P2P координация без центрального сервера. Trustless коммиты: «я обязуюсь участвовать если наберётся N человек». GB синкает состояние подписок в реальном времени. $mol — реактивный счётчик.

**Конкуренты:** Hylo (closest, но централизован), Meetup (не про обязательства). Kickstarter/GoFundMe — про деньги, не действия.

**Вердикт: BUILD** — уникальная ниша, P2P доверие = реальный дифференциатор.

## Tier 3 — Средний фит (3/5)

### 12. Посиделки с экспертом — видеочат клуба (43 коммента)

**Суть:** В клубе много экспертов (кубсаты, микросхемы для космоса), но они не пишут посты — долго. Проще созвониться на час и ответить на вопросы вживую. Нужен инструмент для организации таких встреч: расписание, запись, нотификации.

**Фит:** Scheduling + заметки с сессий через GB. Но сам видеочат — WebRTC/сервер, вне скоупа GB. GB только для мета-слоя.

### 13. Игра «мета-жизнь» со связанными вселенными (22 коммента)

**Суть:** Две вселенные с разными правилами (как игра «Жизнь» Конвея), связанные через несколько пикселей. Интересна с фундаментальной точки зрения: как правила одной вселенной влияют на другую.

**Фит:** Game state через GB, реактивный $mol UI. Но основной риск — геймдизайн, а не технология.
